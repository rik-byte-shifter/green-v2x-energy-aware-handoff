\textbf{Abstract.}\hspace{0.35em}%
This note lists executable steps to reproduce experiments in the \textbf{Green V2X} repository: setup, main suite, validation, weather sweeps, hyperparameter grids, quick checks, tests, and notebooks. Commands assume \textbf{Windows PowerShell} from the \textbf{repository root}. No single script runs everything; order reflects dependencies and cost (heavy jobs last).

\section{Repository root}
All commands assume you start in the project root:
\begin{terminal}
cd "D:\All Projects\Green V2X _Energy Aware Handoff"
\end{terminal}
On other machines, use your clone path. Artifacts default to \path{results/}; some scripts accept \path{--results-dir}.

\section{One-time environment setup}\label{sec:setup}
Create and activate a virtual environment, then install dependencies:
\begin{terminal}
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
\end{terminal}
\begin{tipbox}
\faIcon{leaf}\ \textbf{Tip:} Keep this shell open so \texttt{python} and \texttt{pytest} resolve to the venv for every step below.
\end{tipbox}

\section{Main research suite}
Multi-seed baseline, statistics, plots, scaling, energy-model sensitivity, scenario diversity, and Bangladesh grid-intensity snapshots:
\begin{terminal}
python main.py
\end{terminal}
\begin{infobox}[\faIcon{folder-open}\ Outputs]
Figures and JSON under \path{results/} (e.g.\ \path{simulation_results.json}, scaling and sensitivity JSON, scenario-diversity plots).
\end{infobox}
\begin{infobox}[\faIcon{clock}\ Runtime]
Typically \textbf{tens of minutes to a few hours} on a laptop CPU (hardware \& load dependent).
\end{infobox}

\section{Comprehensive validation}
Literature-scale EPB checks, multi-duration extrapolation, extended CO\textsubscript{2} example, related-work index. Either:
\begin{terminal}
python main.py --comprehensive-validation
\end{terminal}
or:
\begin{terminal}
python validation_runner.py
\end{terminal}
\begin{infobox}[\faIcon{file-alt}\ Artifacts]
\path{results/comprehensive_validation.json}, \path{results/extrapolation_validation.png}.
\end{infobox}

\section{Weather robustness sweep}
Mean and 95\% CIs across seeds for each weather profile:
\begin{terminal}
python weather_sweep.py
\end{terminal}
For geometry aligned with \texttt{paper\_baseline\_scenario} (\textbf{16} BS, \textbf{2800\,m}):
\begin{terminal}
python weather_sweep.py --num-base-stations 16 --area-size 2800 --results-dir results
\end{terminal}
Smoke or alternate output folder:
\begin{terminal}
python weather_sweep.py --num-base-stations 16 --area-size 2800 --results-dir results_smoke
\end{terminal}
Other flags: \texttt{--seeds}, \texttt{--weathers}, \texttt{--duration}, \texttt{--time-step}, \texttt{--num-vehicles}, \texttt{--verbose} (see \path{weather_sweep.py}).

\section{Hyperparameter / connectivity grid sweep}
Full Cartesian product over PHY and energy-aware knobs---\textbf{long} run (\(\sim\)\textbf{45--60\,min} typical laptop, machine-dependent):
\begin{terminal}
python sweep_connectivity_energy.py
\end{terminal}
Writes \path{results/parameter_sweep.json}.

\section{Quick sanity checks}
\begin{terminal}
python verify_fixes.py
python debug_metrics.py
\end{terminal}

\section{Inspect last baseline JSON}
After \texttt{main.py} has written \path{results/simulation_results.json}:
\begin{terminal}
python check_results.py
\end{terminal}

\section{Automated tests}
\begin{terminal}
python -m pytest tests/ -v
\end{terminal}

\section{Optional notebooks}
Jupyter is in \path{requirements.txt}. Explore:
\begin{itemize}
\item \path{notebooks/analysis.ipynb}
\item \path{simulations/Untitled-1.ipynb}
\end{itemize}
These are optional; not part of the scripted pipeline.

\section{Recommended ``run everything'' order}
\begin{enumerate}
\item One-time setup (\S\ref{sec:setup}).
\item \texttt{python -m pytest tests/ -v}
\item \texttt{python verify\_fixes.py} (\texttt{python debug\_metrics.py} optional)
\item \texttt{python main.py}
\item \texttt{python validation\_runner.py} \emph{or} \texttt{python main.py --comprehensive-validation}
\item \texttt{python weather\_sweep.py} (set \texttt{--results-dir} / geometry as needed)
\item \texttt{python sweep\_connectivity\_energy.py} \textbf{last} (largest batch).
\end{enumerate}
Notebooks: any time after setup.

\section{Measuring wall-clock time}
PowerShell:
\begin{terminal}
Measure-Command { python main.py }
\end{terminal}
Swap in \texttt{sweep\_connectivity\_energy.py} or any other script to profile it.
